# handoff — 研发与实测日志

这个目录记录 **Seedance 2.0 提示词方法论是怎么一步步踩坑迭代出来的**，是 `references/` 里那些方法论的「原始证据链」。不影响 Skill 运行，可独立阅读。

## 目录说明

| 子目录 | 内容 |
|:---|:---|
| `creative-plan.md` | 创意规划 |
| `proposals/` | 方法论草案（路径 C v3 升级、路径 D 分镜板驱动） |
| `templates/` | 写 case / summary 用的记录模板 |
| `sessions/` | 按日期归档的工作会话日志 |
| `results/summary.md` | 实测方法论总结（含即梦平台版权审查完整 SOP） |
| `results/cases/` | 30+ 个图生视频实测案例，每个含完整的 v2→v3→v4→v5 迭代轨迹 |
| `results/videos/` | 成片本地存档（占位，实际成片在本地，不入库） |

## 怎么读

- 想看**方法论结论** → 直接读 `results/summary.md`
- 想看**某类场景怎么调** → 翻 `results/cases/` 对应案例
- 想了解**方法论的演进** → 按 `sessions/` 日期顺序读
