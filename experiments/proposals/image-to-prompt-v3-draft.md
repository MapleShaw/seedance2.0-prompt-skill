## 0. 与 v2 的根本差异

| 维度 | v2（当前生效版） | v3（本草案） |
|------|----------------|-------------|
| **图的角色** | 强制作"首/中/尾/反/闪"某一帧锚点 | **降级为"框架参考"，不强制卡帧** |
| **Hook 范式** | 单一"反应模式"（情绪反应+子节拍展开） | **双范式：反应模式 + 凝视模式** |

| **常识审计** | 无 | **必须做日常物理常识检查** |

## 1. 路径 C 的两种 Hook 范式（v3 新增·最重要）

### 1.1 反应模式（v2 已有，保留）

**适用场景**：
- 图本身是"高燃情绪定格"瞬间
- 用户希望主角有明确情绪变化（害羞、惊讶、被击中、笑得失控等）
- 视频核心 Hook 是某个戏剧化的情绪瞬间

**特征**：
- 主体有情绪变化
- 按子节拍展开 ≥X 秒
- 物理时间常数必须遵守
- 时序角色：通常作中间帧/尾帧/反转帧

### 1.2 凝视模式（v3 新增）

**适用场景**：
- 图本身是"日常瞬间"（看球、跟唱、讲课等）
- 用户说"自然神态、不需要发现镜头/慢慢放大/不要切镜/就拍那个 X"
- 视频核心是"被偷拍的日常感"而非戏剧化情绪

**特征**：
- 主体无情绪变化但**自然有动**（捋头发、跟唱、看到精彩动作前倾等）
- 单一缓慢推进，不切不拉远
- 物理时间常数不适用
- 图作首帧或不卡帧
- 美感来源：时间凝固的诗意 + 不知情的真实感

### 1.3 第 1 轮分析阶段必须显式询问用户

```markdown
## 这张图你想要哪种模式？

- `反应` = 主体有情绪反应（害羞/惊讶/被击中/笑），按子节拍展开
  适合：高戏剧性瞬间、明显情绪转折、想做"某一刻的爆发"
- `凝视` = 主体自然存在被偷拍，单一缓慢推进，无情绪变化
  适合：日常瞬间、想做"她/他在做某事的真实记录"
- 不指定 → 我根据图本身判断（默认按图判断戏剧性，平淡日常用凝视，高燃定格用反应）

→ 回复一个字 / 词 即可
```

## 2. 八条核心方法论原则（按重要性排序）

> 完整说明详见 `experiments/summary.md` 第 4 节。本节给出每条的**核心 prompt 模板**。

### ① 图作"框架参考"，不作时间锚点

```
参考@图片1的[主体外貌服装]+[场景氛围]但不强制卡任何一帧，
视频自由演化为日常 [场景类型] 自然瞬间。
```

### ② 凝视模式必须 3-4 个自然小动作

```
主角必须做 [场景] 中自然会做的事（X秒内必须明显发生 3-4 个动作）：
- 动作1（具体动作 + 主动方/被动方）
- 动作2（具体动作 + 反应）
- ...
不可表情呆滞不可一动不动不可只是 [被动姿态] 不可平静
```

### ③ 符合日常物理常识

```
环境：[场景描述]，[关键常识约束]（**禁止 [反常识动作]**）
```

**常识审计 Checklist**（写 prompt 前必过一遍）：
- [ ] 主角的物理位置在该场景下合理吗？（前排不会有人挡）
- [ ] 主角的动作是否物理可行？（不能边面向 A 边对 B 操作）
- [ ] 场景里其他人会做什么？（不能全部静止）
- [ ] 时序逻辑是否符合常识？（讲课要嘴动+看球要有比赛）

### ④ 画质"4 层退化"模板

**通用模板**：
```
画质：[场景对应的录像类型]的双重画质退化：
①[平台压缩 artifacts]——[平台名] 低码率压缩感、画面细节磨平、暗部色块、彩色色噪、压缩边缘抖动；
②[设备瑕疵]——[设备] 在 [环境] 下的 ISO 噪点/曝光起伏/白平衡漂移；
③[现场瑕疵]——[场景特定瑕疵 如 信号瞬抖/雨水痕迹/手肘擦镜]；
④[镜头变形]——广角畸变/色散/脏点污染。
禁用4K高清禁用Cinematic禁用电影级禁用稳定器质感禁用调色精致禁用工业渲染禁用画面锐利禁用清晰画质。
```

**风格锁定结尾必加正向强约束**：
```
画质必须呈现明显的 [X 感] 不可呈现 4K 锐利画质不可呈现电影级稳定画质
```

**4 个场景的"4 层退化"实例库**：

| 场景类型 | 第 1 层（平台压缩） | 第 2 层（设备瑕疵） | 第 3 层（现场瑕疵） | 第 4 层（镜头变形） |
|---------|------------------|------------------|------------------|------------------|
| **手机看直播流** | H.264 低码率块状压缩感 | 手机录屏屏幕摩尔纹+反光+色温偏移 | 直播信号瞬抖让画质瞬降 0.3 秒 | 自动曝光/白平衡漂移+握持抖+脏点 |
| **粉丝手机录夜场** | 抖音/小红书低码率压缩+暗部色块 | 夜场极弱光下感光元件高 ISO 粗粒 | 镜头前雨水痕迹+握持随兴奋节拍抖 | 手机广角畸变+画面边缘色散 |
| **学生手机偷拍课堂** | B 站/视频号低码率压缩 | 教室白光下中等 ISO 粗粒+白平衡略偏冷 | 前景虚焦学生剪影+构图非完美略歪 | 后摄轻微广角畸变+粉笔灰丁达尔 |
| **室内闪光自拍朋友圈** | 微信短视频低码率压缩 | 闪光灯硬光过曝+室内中等 ISO 粗粒 | 镜子反射次级光源+手肘擦镜瞬抖 | 广角自拍轻微鱼眼畸变 |

### ⑤ 群演必须"段内具体写动作"

```
环境同步演化：身边 [群演类型] 必须有可见微动具体写——
左侧 [位置] [群演角色] [具体动作]、
右后方 [群演角色] [具体动作]、
前景 [群演角色] [具体动作]、
远处 [群演角色] [具体动作]。
```

### ⑥ 主角行为"正向夸张 + 反向否定"双重锁定

**模板**：
```
- ✅ [动作 + 幅度强调]（不是 X，是真的在 Y）
```

**例子库**：
| 弱版本（v3 早期） | 强版本（v5 定型） |
|-----------------|------------------|
| "嘴唇随歌词轻动小声跟唱" | "嘴明显张开跟唱嘴动幅度大能看出是在唱（不是抿唇小声哼是真的在唱出声）" |
| "随节拍小幅度晃动" | "头部和身体随 BGM 节拍明显律动+随节拍点头" |
| "主角全程保持自然看比赛状态" | "主角必须有 3-4 个自然小动作分布；不可表情呆滞不可一动不动不可只是站着" |
| 给主角隐含定位 | "她是这条视频的核心主角是全场所有粉丝里最投入的那个" |

### ⑦ 镜头运动锁死声明（凝视模式专用）

```
镜头单一 Very Slow Push In 极缓慢推进（X秒内仅放大约15%），
全程单一机位无任何切换无任何拉远不可上升下降不可切角度。
```

### ⑧ 跨人物互动事件（多人场景必加）

```
跨人物互动事件（视频核心，必须发生）：[发起者] [发起动作] 作为视频起点的招呼信号，
[其他人] [响应动作] 但不需要 [完成度过高的动作]，过程中有自然 [互动类型] 动作发生：
- 角色1：[具体动作 + 对其他人的反应]
- 角色2：[具体动作 + 对其他人的反应]
- ...
```

## 4. 第 2 轮 prompt 必检 Checklist（v3）

输出最终提示词前自动检查：

**核心字段**：
- [ ] **Hook 范式已确定**（反应/凝视）
- [ ] 比例诊断已完成
- [ ] **图作框架参考不卡帧**（除非选择反应模式且图本身就是高戏剧性瞬间）
- [ ] 4 层画质退化已写
- [ ] 9 个禁用清单已写
- [ ] 风格锁定的正向强约束已写（"画质必须呈现 X 感不可呈现 4K"）
- [ ] 建议时长已推算（不可默认 15s）
- [ ] 平台合规风险已评估

**提示词内容**：
- [ ] 前 2 秒有视觉钩子，没有用"缓缓推进"开场
- [ ] X 秒分镜完整，无时间空缺
- [ ] **凝视模式下：主角有 3-4 个自然小动作分布**
- [ ] **凝视模式下：主角行为用"正向夸张+反向否定"双重锁定**
- [ ] **凝视模式下：镜头运动否定句穷举锁死**
- [ ] **多人场景下：必须有跨人物互动事件作为叙事骨架**
- [ ] 群演微动**段内具体写动作**（不只是单句声明）
- [ ] 反应模式下：现实类反应使用了子节拍（≥X 秒）
- [ ] 反应模式下：图作非首帧时显式写了时间锚点
- [ ] 日常物理常识审计已通过（无反常识动作）
- [ ] 文字/UI 内容锁定声明已写
- [ ] 现场音/BGM 贯穿声明已写（"贯穿全片不可寂静不可渐入不可中段才出现"）

## 7. 风险与潜在反对意见

### 7.1 凝视模式可能让"路径 C"边界变模糊

**反对意见**：路径 C 设计初衷是"放大图的潜力"，凝视模式更接近"还原图的氛围"，是否削弱了路径 C 的爆款生产力？

**回应**：实测发现"还原氛围+自然瞬间"反而更接近用户真实需求，爆款不一定要戏剧化。两种范式并存即可，不是替代。

### 7.2 4 层画质退化模板可能过于激进

**反对意见**：所有视频都做"业余偷拍质感"会限制路径 C 在商业向场景的适用性（产品广告、品牌片）。

**回应**：4 层画质退化是**针对特定画质来源**的（生活抓拍/手机偷拍/监控等），商业向场景应保持 v2 原有的"摆拍+电影级"路径。本草案是补充不是替代。

### 7.3 草案完整性

本草案基于 4 个通过验证的 case 沉淀，**未覆盖以下场景**，建议在采纳后通过实战补充：
- 白天日光场景的画质退化模板
- 棚拍/摆拍商业向场景的范式
- 动漫/CG 风格场景下的方法论适配
- 户外日常生活场景（vlog 感）的方法论适配
