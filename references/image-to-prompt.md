# 图片驱动视频提示词生成（路径 C）

> 用户丢一张图进来，希望生成适用于 Seedance 2.0、能在社交媒体引流的视频提示词（v2：时长 4-15s 由图的内容粒度决定，不再默认打满 15s）。可附带文字要求做延展。
>
> 这条路径的核心**不是"配一段提示词"，而是"从一张静态图里识别出动起来最炸的那个点"**。

---

## 适用判断

满足任一条即走路径 C：

- 用户上传/粘贴/拖入一张图片（主要触发条件）
- 用户说"这张图怎么做成视频"、"图生视频提示词"、"用这张图打满15秒"、"根据图片生成视频"等
- 用户在路径 A 进行中突然丢图，则切换到路径 C

**与路径 A/B 的关系：**
- 路径 A（≤15s 概念驱动）：起点是"用户脑里的概念"，做的是收敛创意
- 路径 B（>15s 长视频）：起点是"完整项目企划"，做的是流水线生产
- **路径 C（图片驱动）：起点是"一张已存在的图"，做的是反向放大这张图的潜力**

---

## 核心方法论

### 第一原则：图片比例 = 视频比例（最优解）

否则裁切、变形或黑边几乎不可避免。详见后文"比例处理 5 类策略"。

### 第二原则：一张图必有一个"最有戏的元素"

爆款的本质不是把图均匀地动起来，而是**抓住一个点极致放大**。
读图时强迫自己回答："如果只能让画面里**一个东西**动，应该是哪个？"

### 第三原则：图的角色不止"首帧"一种（必须显式询问用户）

| 图作为 | 演化方向 | 适合什么图 |
|--|--|--|
| **首帧**（0s） | 从图开始演化 | 大部分图（默认） |
| **中间关键帧**（约第 X 秒） | 前面铺垫 → 抵达图 → 余韵 | 图本身就是高燃画面 |
| **尾帧**（最后 0.5-1s） | 故事演化到这张图就结束 | 图是"封面感"的归宿画面 |
| **反转揭示帧**（最后 2-3s） | 看似平凡 → 最后揭示真相 | 图有强反差/震撼感 |
| **闪回/记忆一瞬**（穿插出现 0.5-1.5s） | 主视频是当下，图是过去/未来 | 图是回忆、预兆、对照 |

**强制规则（v2 升级）**：第 1 轮读图分析后**必须显式询问用户图的时序角色**，不能默认按"首帧"处理。

为什么？经过 5 个 case 实测，**当图实际是中间帧但被默认当首帧使用时**：
- 模型自动从图的状态向后演化，前半段就被强行编造
- 视频前后衔接断裂，故事感很差

询问模板见 Step 1.4。**用户选定时序角色后，提示词必须显式标注时间锚点**（例如"第 4.5 秒画面演化抵达 @图片1 所示状态"）。

---

### 第四原则：画质即风格——视频质感必须匹配图的"来源感"（v2 新增）

**反例（5 个 case 实测踩坑）**：图是手机随手拍的生活照，视频却被默认渲染成"电影级 CG 高清稳定"——结果**过于干净反而显假**，用户立刻觉得出戏。

**核心规则**：先判断图的来源类型，再选择对应的画质策略。**默认不要用"电影级稳定"打天下**。

| 图来源线索 | 来源判定 | 视频画质策略关键词 |
|----------|---------|------------------|
| 构图正、光影讲究、后期明显、肤色统一 | **摆拍 / 棚拍** | 电影级稳定、浅景深、调色精致、4K 锐利 |
| 构图随意、光线自然、轻微透视歪斜 | **生活抓拍** | 轻微手抖、自动对焦呼吸、自然曝光起伏、不刻意构图 |
| 低分辨率、噪点明显、色温偏冷或偏黄、白平衡漂移 | **手机日常随手拍** | 数码噪点保留、移动端压缩感、AE/AWB 抖动、握持不稳 |
| 低帧率感、广角畸变、画面有时间戳/GPS 水印 | **监控 / 行车记录仪 / DV** | 低帧率（15-24fps 感）、数码噪点、时间戳、画质压缩感 |
| 颗粒感、暗角、褪色、橙青色偏 | **胶片 / 老照片 / 复古** | 35mm 胶片颗粒、轻微跳帧、画幅切换、年代感色调 |
| 二次元厚涂 / CG 光影 / 非真人 | **动漫 / CG / 立绘** | 保持 CG 质感，允许夸张运镜与超现实物理 |
| 偷拍感、有遮挡物、构图极不正 | **抓拍 / 偷拍** | 第一人称视角、物体遮挡前景、二次对焦、镜头突然甩动 |

**输出落地**：
- 第 1 轮"读图分析"必须输出 **画质来源判定** 字段
- 第 2 轮提示词的"风格总纲"必须包含画质来源对应的关键词，**禁止**默认写"电影级 CG 质感、4K 高清稳定"

**反向清单（生活抓拍类禁止出现的词）**：
- ❌ 电影级、Cinematic、4K 超高清、IMAX
- ❌ 工业光魔级 VFX、Octane 渲染、UnrealEngine5
- ❌ 稳定器质感、超平稳运镜、构图完美
- ✅ 应该写：手机随手拍质感、轻微手抖、自然光、未做后期、像 vlog 抓拍

---

### 第五原则：现实物理时间常数——情绪/反应不能瞬间完成（v2 新增）

**反例**：写"她害羞地脸红了" → 模型给一个 0.3 秒瞬间满脸通红 → 像化妆而不是反应 → 完全出戏。

**核心规则**：现实类视频里的**生理/心理反应**必须按"子节拍"展开，不能只写结果。**动漫/科幻/CG 不受此限**。

| 反应 | 最少占用时长 | 子节拍拆解 |
|------|------------|-----------|
| **害羞脸红** | 2.5s | 眼神飘 0.8s → 低头 / 抿唇 0.5s → 耳根/颧骨缓慢泛红 1.2s（程度有限，不能全脸均匀爆红） |
| **流泪 / 哭** | 3s | 眼眶湿润反光 1s → 眼神颤抖 / 呼吸急 1s → 一滴落下 1s |
| **惊吓 / 一惊** | 1.2s | 瞳孔骤缩 0.2s → 身体微绷 0.3s → 表情变化 0.7s |
| **思考迟疑** | 2s | 停顿 0.5s → 眼神游移 1s → 落定（决定/放弃）0.5s |
| **笑（自然忍不住）** | 2.5s | 眼角先松 0.8s → 嘴角扬起 0.7s → 露齿 / 出声 1s |
| **愤怒 / 怒** | 2s | 屏息 0.4s → 下颌咬紧 / 鼻翼张开 0.6s → 眉头压、眼神冷 1s |
| **悲伤 / 失神** | 3s+ | 眼神涣散 1s → 焦点失去 1s → 嘴唇微动 / 长呼吸 1s+ |
| **羞愤 / 委屈** | 3s | 眼眶微红 1s → 嘴抿紧 1s → 眼泪打转或别开脸 1s |
| **欲言又止** | 2s | 张口 0.3s → 停顿 / 喉结动 0.7s → 闭口低头 1s |

**输出落地**：现实类（人像/生活/纪实）视频提示词中包含上述反应时，**必须写出子节拍**，不能写"她害羞地脸红了"一句话。

**模板**：
```
第 X 秒：[子节拍 1]
第 X+a 秒：[子节拍 2]
第 X+a+b 秒：[最终反应到位]
```

---

### 第六原则：时长服从内容——AI 推算 + 人工确认（v2 新增）

**反例**：所有路径 C 默认"打满 15 秒社交媒体引流" → 一张只够表达"一瞬间反应"的图被硬拉到 15 秒 → 中间 10 秒注水编造 → 故事感崩塌。

**核心规则**：**先判断图适合表达多长的内容，再推算时长**。15s 是上限不是目标。

| 图的内容特性 | 推荐时长 | 典型例子 |
|-------------|---------|---------|
| 一瞬间的反应 / 神态 | **3-5s** | 害羞、惊讶、回眸、一个眼神 |
| 单一动作的完成 | **5-7s** | 喝水、转身、点烟、拨发、撑伞 |
| 一个情绪弧线 | **7-10s** | 迟疑→决定、平静→爆发、克制→释放 |
| 小剧情（两个动作 + 一个反应） | **10-12s** | 进门→看到某物→反应；接电话→脸色变 |
| 完整场景叙事（多动作 + 环境展开） | **12-15s** | 角色在战场行进、产品 360 展示 + 环境爆发 |

Seedance 2.0 family **支持 4-15 秒任意时长**（参数 `--duration=N`），不必硬凑 15。

**输出落地**：第 1 轮读图分析里必须输出：
```
**建议时长**：X 秒
**推算理由**：这张图适合表达[内容类型]，再长会注水/再短会信息不足
```

用户有权否决，但默认按推算值进入第 2 轮和 CLI 调用。

## Step 1：读图分析（路径 C 的核心动作）

### 1.1 必须输出的字段（v2 升级）

按这个结构给用户：

```markdown
## 图片分析

**主体**：[一句话，谁/什么是画面焦点]
**当前画面氛围**：[风格 + 色调 + 情绪]，例如"赛博朋克写实，冷蓝霓虹，孤独"
**最有戏的元素**：[这张图里最适合"动起来"的那个点 + 为什么]

**画质来源判定**（v2 新增）：[摆拍 / 生活抓拍 / 手机日常 / 监控/DV / 胶片复古 / 动漫CG / 抓拍偷拍] —— [一句话依据]
**对应画质策略**：[手机随手拍质感 + 轻微手抖 + 自然曝光 / 电影级稳定 + 浅景深 / ...]（必须写，不要默认电影级）

**比例诊断**：[图片实际比例] → [推荐视频比例]（必要时附说明或扩图建议）

**建议时长**（v2 新增）：[X 秒]
**推算理由**：[这张图适合表达哪种内容粒度，为什么不打满 15s 或为什么需要打满]

**风险检查**：
- [ ] 真人脸：[无 / 有，需要规避方案]
- [ ] 文字/LOGO：[无 / 有，需要裁掉或规避]
- [ ] 清晰度/构图：[OK / 不够，建议先重生一版首帧图]
- [ ] **平台合规风险**（v2 新增）：[无 / 中度（说明哪部分） / 高（拒跑并说明原因）]
```

### 1.2 读图必查的 7 个维度

| 维度 | 看什么 |
|--|--|
| **主体** | 人 / 物 / 场景 / 抽象元素，谁是焦点 |
| **构图** | 主体在画面什么位置（中心/偏左右/上下/分散） |
| **风格** | 写实 / 半写实 / 国漫 / 水墨 / 油画 / 卡通 / 像素... |
| **色调** | 冷暖、明度、饱和度、主色 |
| **质感/材质** | 主体的材质特征（金属/液体/织物/植物/皮肤...） |
| **氛围线索** | 光影、天气、时间、空间空旷度 |
| **隐藏戏点** | 画面里那个"如果它动起来会很炸"的元素 |

### 1.3 风险检查（强制项）

读图时必须扫一遍这三项，发现就告诉用户：

| 风险 | 处理建议 |
|--|--|
| **写实真人脸部** | 平台会拦截。建议：①卡通化/半写实化处理 ②图作为参考但不直接 @引用，改用文字描述 ③用 NanoBanana 重生一版去真人化 |
| **文字 / LOGO / 水印** | 建议裁掉或用 NanoBanana 修掉，否则会被带进视频 |
| **清晰度不足 / 构图差** | 建议先用 NanoBanana 重生一版首帧图，提示词参考 [image-generation.md](image-generation.md) |

### 1.4 图的时序角色询问（v2 新增·强制）

读图分析输出后**必须显式询问用户**，不能默认按"首帧"处理：

```markdown
## 这张图在视频里扮演什么角色？

- `首` = **首帧**（从这张图开始演化，最常用）
- `中` = **中间关键帧**（视频会先铺垫，第 X 秒抵达这张图的状态，再演化下去）
- `尾` = **尾帧**（故事演化到这张图就结束）
- `反` = **反转揭示帧**（前面看似平凡，最后揭示这张图）
- `闪` = **闪回 / 记忆一瞬**（主视频是当下，这张图是过去/未来的画面）
- 不指定 → 我按建议时长选最合适的角色

→ 回复一个字 / 词 即可
```

**用户回复后，第 2 轮提示词必须显式标注时序锚点**，例如：
- 选 `中`：`第 4.5 秒画面演化抵达 @图片1 所示状态`
- 选 `尾`：`最后 1 秒定格在 @图片1 的画面`
- 选 `闪`：`第 6 秒短暂闪回 0.8 秒到 @图片1 的画面，随即恢复主线`

### 1.5 平台合规风险（v2 新增）

Seedance 平台有 **pre-TNS（提交时审）+ post-TNS（生成后审）** 双重审查机制。**post-TNS 失败会扣完积分但平台会退还**——但仍然耗时 5-7 分钟空跑一次。**第 1 轮必须做合规预判**：

| 风险等级 | 触发条件 | 处理 |
|---------|---------|------|
| **高（拒跑）** | 写实/动漫的暴露画面（侧胸 underboob / 高开衩露臀 / 无遮挡半裸 / 透视服装） | 直接告诉用户**不能跑**，建议换图 |
| **中度（警告）** | 紧身贴体服装的特定角度、抽烟 / 酒精消费画面、轻度暴力（持械但无血腥） | 告知有 30-50% 概率被 post-TNS 拦截，让用户决定 |
| **低（继续）** | 服装完整、无政治符号、无血腥、无品牌 LOGO 抢镜 | 正常进入 Step 2 |

**重点**：**动漫风格不豁免合规**。post-TNS 对动漫画面里的暴露同样敏感，**视频化后逐帧审查会比静图严**。

## Step 2：比例处理 5 类策略

**核心原则：让图片比例 = 视频比例，避免任何裁切。**

| 图片比例 | 处理策略 | 默认动作 |
|--|--|--|
| **9:16 竖** | 直接 9:16 视频，最优解（社交媒体原生） | 不询问，直接用 |
| **16:9 横** | 询问用户：①保留 16:9（视频号/B站友好，但抖音/小红书会被压缩）②改 9:16（社交平台原生，但**图会被裁掉左右约 60%**，需主体在中央）③**扩图到 9:16**（推荐，见 Step 5） | 询问 + 推荐扩图 |
| **1:1 方** | 默认 1:1（适合小红书/Instagram）；如用户想做抖音原生竖屏，推荐扩图到 9:16 | 用 1:1 或建议扩图 |
| **接近标准的非常规**（3:4、4:5、2:3） | 明确告知会被归到 9:16 并裁切。建议：①先扩图到 9:16（推荐）②接受裁切（说明哪部分会被裁） | 推荐扩图 |
| **极端比例**（2.35:1、超长条、超宽幅） | **不建议直接当首帧**。备选：①作为"画中画"嵌入到 9:16/16:9 的新构图里 ②先扩图到标准比例 ③仅作为风格参考，不直接 @引用 | 强制不直接用 + 给备选 |

**用户用文字明确指定比例时（"我要竖屏"），文字优先级 > 图片比例。**

## Step 3：Hook 模式匹配（图特征 → 推荐模式）

读图后根据"主体类型 + 最有戏的元素"，从下表选 2-3 个候选模式。

### 模式速查表

| 图特征 | 推荐 Hook 模式 | 默认演化方向 |
|--|--|--|
| **有强材质感的物体**（玫瑰、金属、果实、玻璃...） | **材质魔术** | 触发 → 材质瞬变（结晶/液化/燃烧/汽化） |
| **人像 / 角色立绘** | **风格突变** / **服装/造型变化** / **情绪演绎** | 写实↔水墨/油画/像素 / 100年挑战变化 |
| **风景 / 场景 / 建筑** | **一镜到底（拉远/潜入）** / **时间流逝** | 微观→宏观 / 白昼→黑夜 / 完好→废墟 |
| **产品 / 商品图** | **360 展示 + 环境爆发** / **材质魔术** | 旋转展示叠加粒子/光效爆发 |
| **微观元素**（花瓣、水珠、晶体） | **尺度穿越** | 微距→星云→宇宙的递归 |
| **强构图人物动作图** | **子弹时间 / 定格环绕** | 360 环绕被定格的瞬间 |
| **有反差感 / 荒诞感的图** | **2 秒反转** | 看似正常 → 最后揭示真相 |
| **抽象 / 流体 / 粒子图** | **极致满足感** / **完美循环** | 流畅生长/塑形/绽放 |
| **赛博朋克 / 科幻 / 末世场景** | **史诗大制作** | 调用品质锚定+光影三层+大气连贯 |

### 输出 Hook 候选时的格式

每个候选必须包含：

```markdown
### 模式 [A/B/C]：[模式名称]
- **演化轨迹**：基于推算时长 X 秒分镜（必须明确每段时间和动作；X = Step 1 推算值，4/6/8/10/12/15 任选）
- **图的时序角色**：首帧 / 中间关键帧（第 K 秒） / 尾帧 / 反转揭示帧 / 闪回（必须与 Step 1.4 询问结果一致）
- **传播因子**：[这个版本的钩子是什么——为什么用户会想分享/二刷]
- **适用度**：★★★★★（一句话理由）
```

### 数量策略

- **默认给 2 个版本**（最稳的 + 一个差异化的）
- **当判断这张图作为首帧"演化空间有限"时**（比如已经是高燃画面），追加一个"中间关键帧"或"反转帧"备选
- **不要每次都强行 3 版**，避免决策疲劳
- **候选模式必须基于 Step 1.4 用户选定的时序角色**（v2）；用户选 `中` 就主推模板 B 系列，选 `反` 就主推模板 C，选 `不指定` 才你自由发挥

## Step 4：演化模板库（按图的时序角色 + 推算时长）

按"图作为什么角色"提供模板。所有模板都遵守：
- **前 2 秒生死线**：开场必须有视觉冲击或悬念
- **一个视频一件事**：单一核心概念
- **时长由 Step 1 推算确定**（v2 改）：4 / 6 / 8 / 10 / 12 / 15s 任选
- **画质来源已声明**（v2 新增）：在风格总纲里写明对应来源的画质策略，禁止默认电影级
- **现实物理时间常数已遵守**（v2 新增）：现实类视频里的情绪反应必须按子节拍展开

### 模板 A：图作为首帧（默认，最稳）

```
[X秒][风格总纲，含画质来源关键词]，
0-2秒：@图片1为首帧静态画面，[微动铺垫——如尘埃浮动/光线变化/角色呼吸]，
为下一刻爆发蓄势；
3—（X-3）秒：[核心戏点爆发——材质瞬变/动作展开/镜头推进/风格突变启动]，
[运镜：组合 1-2 个]；
（X-2）—X 秒：[余韵收束——画面凝固成新的"封面感"，或回呼应开头]。

风格锁定：[一句风格描述，含画质来源]，禁止任何文字、字幕、LOGO、水印。
```

> X = 推算时长（4/6/8/10/12/15s），中段长度按比例分配。

### 模板 B：图作为中间关键帧（v2 改名，原"高潮帧"）

```
[X秒][风格总纲，含画质来源关键词]，
0—（K-1）秒：[平静铺垫——主体的局部特写/前情交代，与最终画面有视觉关联]，
[运镜：缓慢推进或微动]；
（K-1）—K 秒：画面演化抵达 @图片1 所示状态，[关键帧定格 0.5 秒]，
[运镜：定住或环绕]；
K—X 秒：[超越关键帧的余韵——画面继续延伸/碎散/回落]，
[音效：高点之后的回声]。

风格锁定：[一句风格描述]，禁止任何文字、字幕、LOGO、水印。
```

> X = 总时长（推算值），K = 抵达图的时刻（默认 X×0.55，例如 X=10 时 K≈5.5s）。
> **关键**：必须在提示词里显式写"第 K 秒画面演化抵达 @图片1 所示状态"，模型才不会乱演化。

### 模板 C：图作为反转揭示帧（最后 2-3s）

```
[X秒][风格总纲，含画质来源关键词]，
0—（X-4）秒：[看似平常的开场——刻意营造"这就是个普通xx"的预期]，
[运镜：标准、克制]；
（X-4）—（X-2）秒：[转折触发——某个动作/视角变化让一切开始改变]；
（X-2）—X 秒：[揭示——画面演化为 @图片1 所示状态]，
[音效：突然的静默或一记重音]。

风格锁定：[一句风格描述]，禁止任何文字、字幕、LOGO、水印。
```

### 模板 D：图作为尾帧（v2 新增）

```
[X秒][风格总纲，含画质来源关键词]，
0—（X-2）秒：[完整的故事推进——铺垫 → 发展 → 接近图的状态]，
[运镜：依故事节奏]；
（X-2）—X 秒：画面定格成 @图片1 所示状态，
[最后一秒静止 / 极慢推进 / 暗角渐入]。

风格锁定：[一句风格描述]，禁止任何文字、字幕、LOGO、水印。
```

### 模板 E：图作为闪回 / 记忆一瞬（v2 新增）

```
[X秒][风格总纲，含画质来源关键词]，
0—K 秒：[当下时间线的主画面——主体的当前状态]，
[运镜：与当下场景一致]；
K—（K+0.8）秒：[短暂闪回 / 一闪而过——画面快速切换为 @图片1 所示状态]，
[过渡：白光闪 / 模糊抽帧 / 心跳声切入]；
（K+0.8）—X 秒：[回到当下时间线，主体表情或环境因为闪回而有微妙变化]。

风格锁定：[一句风格描述]，禁止任何文字、字幕、LOGO、水印。
```

> K = 闪回触发点（默认 X×0.4），闪回本身只占 0.5-1.5s。

### 模板填充时的注意事项（v2 升级）

- **时长 X 由 Step 1 推算确定**，不再默认 15s
- **风格总纲必须包含画质来源关键词**（Step 1.1 输出的"对应画质策略"）
- **现实类视频中的情绪反应必须用子节拍**（参见第五原则的反应时间常数表）
- **每段都要有"画面 + 镜头"**，可选叠加"音效"
- 提示词总长度建议 ≤120 字（核心部分≤60 词的精神）；过长会稀释精度
- 运镜词参考 [vocabulary.md](vocabulary.md)
- 风格锚定参考 [image-generation.md](image-generation.md)

## Step 5：扩图建议（增值能力）

当图片比例不理想，但构图本身很好时，主动给用户扩图方案。

### 触发条件（满足任一）

- 图是 16:9 / 1:1 / 非标准比例，但用户想做 9:16 抖音视频
- 图主体居中、左右/上下有可延伸的氛围空间
- 用户明确希望"打满 9:16"获得最大社交媒体曝光

### 扩图工具推荐

- **NanoBanana（默认）**：自然语言描述扩展方向，对中文友好
- **即梦图片生成**：可指定"扩图"功能，但写实人脸场景受限
- **Photoshop 生成式填充**：精度最高，但需要本地操作

### 扩图提示词模板（NanoBanana）

```
[原图描述：主体+风格+色调]，将原画面[扩展方向：上方/下方/左右]扩展，
扩展区域延续原画面的[风格/光影/氛围]，[扩展区域具体内容描述：天空/地面/环境延伸]，
保持主体不变，整体画面比例[目标比例：9:16竖版/16:9横版/1:1方形]，
风格统一无拼接痕迹。禁止：任何文字、字幕、LOGO、水印。
```

### 输出扩图建议时的格式

```markdown
### 扩图建议（推荐）

**原图比例**：16:9 横版
**目标比例**：9:16 竖版（抖音原生）
**扩展方向**：上方 + 下方
**扩展内容建议**：
- 上方：延续画面的天空/光线，扩出 [具体描述]
- 下方：延续画面的地面/前景，扩出 [具体描述]

**扩图提示词**（NanoBanana）：
\`\`\`
[完整可复制的扩图提示词]
\`\`\`

扩完之后再用扩图后的 9:16 图作为首帧 @图片1 接入下方视频提示词。
```

## Step 6：文字要求注入规则

用户附带的文字要求，按 4 级优先级处理（高到低）：

### Level 1：硬约束（必须满足，否则方案废）

例：
- "不要出现人脸"
- "必须 9:16 竖屏"
- "不要任何文字"
- "时长必须 15 秒"
- "不要某品牌/角色"

→ **直接进生成参数或负面提示词**，不询问，不妥协。

### Level 2：延展方向（覆盖图片本身的风格判断）

例：
- "往赛博朋克方向"
- "做成东方仙侠"
- "改成水墨风"
- "现代都市感"

→ **覆盖**我从图里读到的风格判断，**自动激活"风格突变"模式作为推荐 A**（因为图本身风格 ≠ 用户要的风格，本来就该走风格突变能力）。

### Level 3：情绪 / 用途（影响 hook 选择和收束）

例：
- "做奶茶店开业宣传"
- "想要高级感"
- "要治愈系"
- "用来涨粉"

→ **影响 hook 模式排序**和**结尾收束方式**。比如奶茶店开业 → 优先"极致满足感 + 产品 360 展示"。

### Level 4：风格细节（注入到具体段落）

例：
- "加点慢镜头"
- "用航拍视角"
- "加雨水特效"

→ **注入到对应分镜的运镜或大气描述**。

### 冲突处理

| 冲突类型 | 处理方式 |
|--|--|
| 用户文字 ≠ 图本身风格 | 风格突变模式自动成为推荐 A，告诉用户"这本来就是 Seedance 的强项" |
| 用户要"15s 满" + 图作为反转帧 | 优先满足 15s，反转帧版本的 0-12s 用模板 C 的铺垫填充 |
| 用户要"竖屏" + 图是 16:9 | 优先满足竖屏，主推扩图方案（不裁切原图） |

## 完整交互流程（v2 升级）

```
用户：[丢图 + (可选)文字]
  ↓
Skill：执行 Step 1（读图分析） + Step 2（比例诊断）
  - 输出含画质来源判定 / 建议时长 / 平台合规风险
  ↓
Skill：Step 1.4 询问图的时序角色（首/中/尾/反/闪）
  ↓
用户：回复时序角色（或不指定）
  ↓
Skill：基于时序角色 + 推算时长，输出 2-3 个 Hook 候选模式（Step 3）
  ↓
用户：选 A/B/C 或自己提方向
  ↓
Skill：基于选定模式 + 对应模板（Step 4）+ 文字要求（Step 6）
  - 风格总纲带画质来源
  - 现实类反应用子节拍
  - 时长 = 推算值（非默认 15）
  ↓
输出完整可复制的 Seedance 提示词
  ↓
（可选）追加 CLI 执行询问，参数含推算时长，详见 cli-integration.md
  ↓
（可选）用户要求微调 → 在选定版本上调整
```

## 路径 C 的质量自检 Checklist（v2 升级）

输出最终提示词前，自动检查：

**核心字段**：
- [ ] **比例诊断已完成**，图片比例与视频比例的匹配方案明确
- [ ] **图的时序角色已询问并确定**（首/中/尾/反/闪）（v2 升级）
- [ ] **画质来源已判定**，对应画质策略已写入风格总纲（v2 新增）
- [ ] **建议时长已推算**，并给出推算理由（v2 新增）
- [ ] **平台合规风险已评估**（高 → 拒跑 / 中 → 警告 / 低 → 继续）（v2 新增）
- [ ] **风险检查已扫**（真人脸/文字水印/清晰度）

**提示词内容**：
- [ ] **前 2 秒有视觉钩子**，没有用"缓缓推进"开场
- [ ] **X 秒分镜完整**，无时间空缺（X = 推算时长，不是默认 15）（v2 改）
- [ ] **画质来源关键词已注入**风格总纲（v2 新增），且**未误用**电影级关键词在生活类图上（v2 新增）
- [ ] **现实类反应使用了子节拍**（脸红 ≥2.5s、哭 ≥3s、惊吓 ≥1.2s 等）（v2 新增）
- [ ] **图作为非首帧时**，提示词中显式写明时间锚点（如"第 X 秒抵达 @图片1 状态"）（v2 新增）
- [ ] **@图片1 引用方式明确**（首帧 / 编辑 / 参考）
- [ ] **风格锁定语句已加**（防漂移）
- [ ] **文字要求已分级注入**（硬约束/延展/情绪/细节）
- [ ] **扩图建议已给出**（比例不理想时）
- [ ] **二刷钩子已设计**（结尾呼应或细节）
- [ ] **总文件数 ≤12**（图+视频+音频）
- [ ] **未包含写实真人面部素材**

## 常见误区（v2 升级）

1. **平均用力让所有元素都动**：错。一张图只放大一个戏点。
2. **直接把图比例当视频比例不询问**：错。16:9 横图做抖音是高频踩坑。
3. **看到漂亮图就默认作首帧**（v2 强化）：错。**必须显式询问用户图的时序角色**，不能默认。
4. **文字要求一股脑塞 prompt**：错。必须分级注入。
5. **扩图建议只提一句**：错。要给出完整的扩图提示词，让用户能直接执行。
6. **跳过风险检查**：错。真人脸/水印/清晰度/合规风险不查清楚，跑完发现被拦截就晚了。
7. **画质默认电影级 CG 高清稳定**（v2 新增）：错。**生活类图变高清就是变假**。视频画质必须匹配图的来源感（手机/抓拍/监控/胶片/CG 各有策略）。
8. **写情绪反应只写结果**（v2 新增）：错。"她害羞地脸红了"会被模型瞬间执行，必须按子节拍展开（眼神飘 → 低头 → 缓慢泛红）。
9. **默认打满 15s**（v2 新增）：错。**时长服从内容**。一瞬间的反应 3-5s 即可，硬拉到 15s 必然注水编造。
10. **动漫风格 = 合规豁免**（v2 新增）：错。**post-TNS 对动漫暴露同样敏感**，视频化逐帧审查比静图严。
